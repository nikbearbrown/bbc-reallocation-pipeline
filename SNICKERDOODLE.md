# BBC Reallocation Pipeline — Constitution

## Prime Directive
Use verified local data and tested scripts first. Use an LLM only to explain, summarize, draft, or make bounded judgments after data has been checked. Every claim needs evidence.

## Core Principle
This system protects Nik's attention. It runs silently. It surfaces only when an opportunity clears all gates without human input, the fellow match is unambiguous, and the pay is worth a context switch from teaching and Pokemon.

## Data Privacy
All fellow personal data lives in private/bbc/ which is gitignored. Never commit, paraphrase, or log personal data, contact details, or private application notes. npm run doctor enforces this.

## Two Sourcing Streams
Stream 1: Automated platform scanning — runs silently, feeds combiner
Stream 2: Fellow-sourced — fellow submits URL, system gates it, surfaces in BBC_DAILY_BRIEF if it clears

## Interrupt Threshold
The system messages Nik only when ALL of the following clear:
- BBC net hourly ≥ $15/hr
- Fellow match is unambiguous (2+ verified skills, hours available)
- Scope is defined (deliverable named, not "help us with AI")
- At least 1 hire-intent signal
- Nik's required time: approval only, 10 minutes max

## Fellow Intake Rule
Fellows may submit postings via Claude. The system runs the same gate logic. Rejections include specific feedback so fellows learn to pre-filter. Approvals surface in BBC_DAILY_BRIEF labeled "fellow-sourced."

## Notification Interface
BBC_DAILY_BRIEF: Nik's daily Cowork conversation — brief, scannable, yes/no decisions only
BBC_FELLOW_BRIEF: Fellow's Claude conversation — approved opportunities matching their stack only
